Основной особенностью (killer feature) git является ветвление.

	Основной веткой проекта в git является master. Она создается после инициализации git репозитория.
	Для создания ответвления необходимо выполнить команду git checkout -b <branch name> и является сокращением двух команд git branch <branch name> и git checkout <branch name>, которые нужны для создания ветви и перехода в нее соответственно.

	При работе с ветками появляется необходимость слияния нескольких разных веток. Для этого из ветки в которую выполняется слияние выполняется команда git merge <branch name>. 
В случае конфликта необходимо отредактировать конфликтные файлы вручную или выбрать другой метод разрешения конфликтов при слиянии.
Существует и другой способ включить изменения из одной ветки в другую - перемещение.
Выполняется командой git rebase <branch name>. При помощи этой команды вы можете взять все изменения, которые попали в коммиты на одной из веток, и повторить их на другой.

	Для отправки локальных веток в удаленный репозиторий необходимо использовать выполнить команду git push <repository adress> <remote branch>, для установки связи между ними необходим параметр -u.
Для того чтобы отслеживать удаленные ветки необходимо выполнить команду git checkout --track <repository address>/<remote branch>.
Для удаления удаленных веток необходимо выполнить команду git push <repository address> :<remote branch>.
