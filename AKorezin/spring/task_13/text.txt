Эссе по процессам и потокам.

	Процесс - это выполняющийся экземпляр компьютерной программы. Процесс может состоять из нескольких потоков, которые выполняются параллельно. В операционной системе для выполнения несольких процессов одновременно применяется многозадачность, когда процессорное время делится между разными процессами. Так как на одном процессоре(одном ядре) может выполняться одновременно только один процесс, то для обеспечения многозадачности процессор переключается между разными процессами, останавливая один и запуская другой.
	В общем случае процесс состоит из:
	- образа исполняего машинного кода программы
	- памяти, которая содержит исполняемый код, входные и выходные данные, стек вызова и кучу для хранения данных вычислений
	- дескрипторов ресурсов операционной системы, которые выделяются для процесса
	- параметров безопасности, такие как владелец процесса и разрешенные операции
	- состояния процессора, который содержит регистры, адреса физицеской памяти. Когда процесс выполняется состояние хранится в регистрах, а когда не выполняется - в памяти.

	Поток - это часть процесса, которая может выполняться отдельно. Несколько потоков одного процесса могут разделять ресурсы, в то время, как разные процессы этого делать не могут. На одном процессоре многопоточность обычно разделением процессорного времени между потоками и работает также как и многозадачность, то есть потоки выполняются псевдопараллельно. На многопроцессорных и многоядерных системах потоки могут выполняются параллельно и одновременно. Каждый процессор или ядро может выполнять свой собственный поток.
	Отличия процесса от потока:
	- процессы обычно независимы, а потоки существуют только внутри процесса
	- процессы имеют разделенное адресное пространство, а потоки делят свое адресное пространства
	- процессы взаимодействуют друг с другом только через специальный механизм межпроцессного взаимодействия, обеспечиваемый операционной системой
	- переключение между потоками выполняется быстрее, чем переключение между процессами

	Для взаимодействия потоков используются:
	- мьютексы
	- семафоры
	- события
	- критические секции
	- условные переменные.
	Мьютекс(взаимоисключение) - это объект синхронизации, который устанавливается в особое состояние сигнала, когда не занят каким-либо потоком. Только один поток может владеть мьютексом в любой момент времени. Одновременной доступ к общему ресурсу исключается. Применяется для исключения одновременной записи с нескольких потоков в общий участок памяти.
	Семафоры - это объект, который ограничивает количество процессов, которые могут войти в заданный участок кода. Это логическое расширение мьютекса - семафор со счетчиком 1 эквивалентен мьютексу.
	Событие - это объект, который имеет два состояния: просигнализирован или нет, над которым опредены операции: просигнализировать, сбросить в непросигнализированное состояние и ожидать.
	Критическая секция обеспечивает синхронизацию подобно мьютексу за исключением того, что объекты, представляющие критические секции, доступны в пределах одного процесса.
